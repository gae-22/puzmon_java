# ゲーム管理 - Main, GameData

## Main クラス

ゲーム全体の初期化と進行を管理するメインエントリーポイント。

### クラス概要

```java
public class Main
```

ゲームの起動、初期化、ゲームループの実行、終了判定を行います。

### メソッド

#### `public static void main(String[] args)`

ゲームのメインメソッド。プレイヤーの入力を受け付け、複数の敵とのバトルを進行させます。

**処理フロー**

1. 出力プロバイダの初期化
2. コマンドリーダーの構築
3. プレイヤー名の入力
4. パーティ情報の表示
5. 敵との順次バトル処理
6. パーティ全滅またはすべての敵を倒すまでループ
7. 結果表示（クリア/ゲームオーバー）
8. 倒したモンスター数を表示

**パラメータ**

- `args`: コマンドライン引数（使用されない）

**処理例**

```
プレイヤー名を入力
→ パーティ編成表示
→ 敵1とバトル
→ 敵1撃破
→ 敵2とバトル
...
→ すべての敵を撃破
→ "GAME CLEARED!!" 表示
```

## GameData クラス

ゲーム全体の定数とデータ生成を管理する不変クラス。

### クラス概要

```java
public final class GameData
```

ゲームの定数、敵・プレイヤーのモンスターデータ生成を行う。

### 定数

#### `public static final int GEMS_LENGTH = 14`

ジェムボード上のジェムの個数（A～N の 14 個）。

#### `public static final int PARTY_SIZE = 4`

パーティに属するモンスターの最大数。

#### `public static final int LINE_LENGTH = 29`

画面表示時の区切り線の長さ。

### メソッド

#### `public static Monster[] createEnemies()`

敵モンスターの配列を生成する。

**戻り値**
敵モンスターの配列。以下の 5 体を順序固定で返す：

1. スライム（水、HP=100、攻撃=10、防御=1）
2. ゴブリン（土、HP=200、攻撃=20、防御=5）
3. オオコウモリ（風、HP=300、攻撃=30、防御=10）
4. ウェアウルフ（風、HP=400、攻撃=40、防御=15）
5. ドラゴン（火、HP=600、攻撃=50、防御=20）

**処理例**

```java
Monster[] enemies = GameData.createEnemies();
// enemies.length == 5
// enemies[0].getName() == "スライム"
```

#### `public static Monster[] createPlayers()`

プレイヤーのモンスター配列を生成する。

**戻り値**
プレイヤーモンスターの配列。以下の 4 体を順序固定で返す：

1. 朱雀（火、HP=150、攻撃=30、防御=10）
2. 青龍（風、HP=150、攻撃=20、防御=10）
3. 白虎（土、HP=150、攻撃=25、防御=5）
4. 玄武（水、HP=150、攻撃=25、防御=15）

**処理例**

```java
Monster[] players = GameData.createPlayers();
// players.length == 4
// players[0].getName() == "朱雀"
```

### ゲーム進行の流れ

1. **初期化段階**
    - `GameData.createPlayers()` でプレイヤーパーティを作成
    - `Party` にプレイヤー名を設定
    - パーティ情報を表示

2. **バトルループ**
    - `GameData.createEnemies()` から敵を 1 体ずつ取得
    - 各敵に対して `BattleManager` でバトル実行
    - 敵を全滅させるまたはパーティが全滅するまで継続

3. **終了判定**
    - すべての敵を倒せばゲームクリア
    - パーティが全滅すればゲームオーバー
