# Puzzle & Monsters - 完全設計ドキュメント

このディレクトリには、Puzzle & Monsters ゲームの包括的な設計・仕様ドキュメントが保存されています。

## ドキュメント一覧

### 1. **GAME_DOCUMENTATION.md** - ゲーム設計の基礎

**対象読者：** 全員（プレイヤー、開発者、アーキテクト）

ゲーム全体の概要から詳細なクラス設計まで、すべての基本情報を網羅しています。

**主な内容：**

- ゲーム概要と特徴
- ゲームの流れ（フローチャート）
- 12個のメインクラスの詳細説明
    - 各クラスのプロパティ
    - 各メソッドの役割
    - プレイヤー/敵モンスターのデータ
- システム設計（クラス関係図）
- 属性システム（相性表を含む）
- ダメージ計算システム
- ジェムボードメカニクス

**セクション別ガイド：**

```
1. ゲーム概要          →ゲームの基本的な説明
2. ゲームの流れ        →全体的なゲーム進行図
3. 主要クラス説明      →各クラスの深い理解に必須
4. システム設計        →アーキテクチャの理解
5. 属性システム        →相性判定の仕組み
6. ダメージ計算システム →計算式と例
7. ジェムボードメカニクス →パズルの仕組み
```

**読むべき順序：**

1. 「ゲーム概要」でゲーム全体を理解
2. 「ゲームの流れ」で進行順序を把握
3. 「主要クラス説明」で各要素を詳しく学ぶ
4. 「ダメージ計算」「属性システム」で複雑な仕組みを理解

### 2. **SYSTEM_DIAGRAMS.md** - ビジュアル可視化

**対象読者：** 開発者、アーキテクト、ビジュアル学習者

図表を用いてシステムの複雑な関係を視覚化しています。Mermaid ダイアグラムで描画。

**主な図表：**

- **クラス構成図** - すべてのクラスと関係を表示
- **クラス依存関係図** - クラス間の依存性フロー
- **バトルフロー図** - ゲーム全体の処理フロー
- **ジェムマッチプロセス** - マッチ判定から表示までの流れ
- **ダメージ計算フロー** - ダメージ計算の各要素
- **ゲーム初期化シーケンス** - オブジェクト生成の順序（シーケンス図）
- **単一ターン処理シーケンス** - 1ターン内のすべての処理（シーケンス図）
- **属性相性図** - 属性相性の視覚化
- **相性倍率マトリックス図** - 相性倍率の行列表示
- **パーティHP管理** - HP変化の流れ
- **コンボシステム** - チェーン処理の仕組み

**用途：**

- 処理フローの全体像把握
- クラス関係の複雑性の理解
- デバッグ時の流れの追跡

**アクセス方法：**
VS Code の Mermaid Preview 拡張機能で図表を直接閲覧できます。

## 🔑 主要概念のクイックリファレンス

### ゲームの勝利条件

```
5体の敵をすべて倒す
→ ゲームクリア
```

### ゲームの敗北条件

```
パーティの総HP が 0 以下になる
→ ゲームオーバー
```

### 属性相性の基本ルール

```
火 → 風を倒せる（2倍ダメージ）
風 → 土を倒せる（2倍ダメージ）
土 → 火を倒せる（2倍ダメージ）
```

### ダメージ計算式

```
最終ダメージ = 基本 × 相性 × コンボ × ランダム
（最小値は1）
```

### コンボ倍率

```
コンボが多いほど攻撃力が増加
基本倍率: 1.5^n (n = ジェムマッチ数 - 3 + コンボ数)
```

### ジェムマッチ条件

```
同じ属性のジェムが「3個以上連続」で配置
→ マッチ判定
```
